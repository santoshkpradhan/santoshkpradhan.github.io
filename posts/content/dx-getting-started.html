I am writing a multi-part article each covering different areas of Salesforce DX. In this one, I would like to cover

  • Working with your existing Salesforce Orgs
As a beginner, the concept of Scatch org might sound overwhelming. So, forget about scratch orgs. Lets get started with whatever currennt org structure we have.

This can be divided into following steps:

  • Creating your Project
  • Authenticating your Org

Create Project

Open VS Code, and open the console in VS Code. To open the console you could use hotkeyctrl + ~. In the console, you type the following command to create your project
sfdx force:project:create --projectname dx-getting-started. Then you see following set of files created

target dir = /Users/santoshpradhan/Desktop/GlobalDrive/Development/dx/dx-getting-started create dx-getting-started/sfdx-project.json create dx-getting-started/README.md create dx-getting-started/.forceignore create dx-getting-started/config/project-scratch-def.json

Authenticate Salesforce Org

Once you executed command above, you should see a list of files generated and one of them issfdx-project.json.

If you want to connect to a Production or Developer Edition Org, you could use sfdcLoginUrlas login.salesforce.com. If you want to connect to Sandbox org, you could set the Url astest.salesforce.com.
Now your settings are done. You are good to go ahead with authentication.

To open the authentication window, run the following command in the console of VS Code.

sfdx force:auth:web:login

Then the login screen opens and once you enter the credentials, on successful login, comeback to your VS Code window. You can see a message in the console something like this:
Successfully authorized pradhan.santosh@***.com with org ID 00D**********. You may now close the browser
